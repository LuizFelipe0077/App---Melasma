The Deployment Bible (Módulo 21)

O Manual Definitivo e Interativo de Implantação e Operação do Sistema de Acompanhamento Clínico Integrativo. Siga as instruções cronologicamente.

1. Visão Geral da Arquitetura

O sistema utiliza uma arquitetura "Serverless No-Code DB". O código frontend vive no GitHub Pages, o backend executa no ecossistema serverless do Google e os dados persistem em planilhas.

2. Pré-requisitos

🔧 Ferramentas Necessárias
  • Conta Google: Onde o banco (Sheets) e a API (Apps Script) ficarão hospedados.
  • Conta GitHub: Onde o código-fonte ficará versionado e o Frontend será hospedado.
  • Git Instalado: Para subir o código para o GitHub (git-scm.com).
  • Visual Studio Code (VSCode): Editor de código.
  • Navegador Chrome/Edge: Recomendado para debugar.

3. Configurando o Google Sheets (Banco de Dados)

Configurar o Sheets é o primeiro passo técnico, pois precisaremos do ID da planilha para os passos seguintes.

3.1. Criação da Planilha

  1. Acesse sheets.google.com.
  2. Crie uma nova planilha "Em branco".
  3. Renomeie para: DB_CLINICA_PRODUCAO.
  4. Na URL do navegador (ex: https://docs.google.com/spreadsheets/d/ID_AQUI/edit), copie a sequência do ID_AQUI. Guarde-o.

3.2. Abas Obrigatórias

Crie exatamente estas abas (respeite as letras maiúsculas/minúsculas):

  • Pacientes
  • Protocolos
  • Suplementos
  • Check_Ins
  • Gamificacao
  • Permissoes
🔒 Segurança (Importante)

Clique em Compartilhar no canto superior direito e garanta que o Acesso Geral seja Restrito (Apenas a sua conta). O Frontend nunca acessa a planilha diretamente, apenas através do backend.

4. Configurando o Google Apps Script (Backend)

Aqui colocaremos a inteligência da aplicação.

  1. Na sua planilha recém-criada, clique em Extensões > Apps Script.
  2. Renomeie o projeto para API_CLINICA_PRODUCAO.
  3. Copie todos os arquivos da pasta backend/src/ do projeto para dentro do editor. Você deve criar arquivos no editor (ex: GasController.gs) e colar o conteúdo do correspondente .js local neles.
⚙️ Configurando o ID do Banco

Encontre o arquivo SystemConfiguration.gs e altere a seguinte linha com o ID do Sheets copiado no Capítulo 3:

static DATABASE_SPREADSHEET_ID = 'COLE_O_ID_DA_SUA_PLANILHA_AQUI';

5. Publicando o Backend (Web App)

Iremos transformar o Apps Script em uma API pública que recebe requisições JSON.

  1. No editor Apps Script, clique no botão azul Implantar > Nova implantação.
  2. Selecione o tipo App da Web (ícone de engrenagem).
  3. Descrição: Versão 1.0.0.
  4. Executar como: Eu (Seu e-mail). (Dá permissão ao script para editar sua planilha).
  5. Quem pode acessar: Qualquer pessoa. (Crítico para que o Frontend consiga comunicar sem bloqueio do Google).
  6. Clique em Implantar. Autorize as permissões na tela do Google.
  7. Copie a URL do App da Web (Ela termina com /exec).

6. Conectando o Frontend

Agora dizemos ao código local para onde enviar os dados.

  1. Abra o VSCode no seu computador.
  2. Vá até frontend/src/infrastructure/api/ApiClient.js.
  3. Localize a URL base e substitua pela URL do /exec gerada no passo anterior.
const GAS_URL = "https://script.google.com/macros/s/SUA_URL_COPIADA/exec";

7. Publicando o Frontend (GitHub Pages)

Iremos colocar a interface gráfica na internet.

  1. No terminal, inicialize o repositório git e envie para o GitHub:
    2. git init
git add .
git commit -m "Go Live"
git branch -M main
git remote add origin https://github.com/SEU_USUARIO/clinica-app.git
git push -u origin main
  2. Acesse o repositório no GitHub.com. Vá em Settings > Pages.
  3. Em Source, escolha Deploy from a branch.
  4. Selecione a branch main e clique em Save.
  5. Em 2-3 minutos, sua URL pública estará disponível (ex: https://usuario.github.io/clinica-app/).

8. Testando o Paciente Zero

Como logar pela primeira vez no sistema recém-criado.

  1. Acesso Admin: Como ainda não há tela de cadastro público, vá na sua planilha DB_CLINICA_PRODUCAO, aba Permissoes.
  2. Escreva manualmente na linha 2: Seu Email, Uma senha com Hash Bcrypt, e a Role ADMIN.
  3. Abra o URL do seu GitHub Pages e faça login.
  4. Na Dashboard, clique em Novo Paciente. Preencha nome e senha temporária (ex: "123456").
  5. Prescreva 1 protocolo e 1 suplemento.
  6. Deslogue, e logue como o Paciente. Veja a mágica acontecer!

9. Troubleshooting (Problemas Comuns)

⚠️ Erro CORS durante o Login

Causa: Apps Script publicado incorretamente.

Solução: Vá no Apps Script > Nova Implantação. Mude "Quem pode acessar" de 'Apenas minha organização' para 'Qualquer pessoa'. Atualize a URL no `ApiClient.js` e faça push pro GitHub.

⚠️ Tela Branca: "Unexpected token < in JSON"

Causa: A URL da API está exigindo login do Google ou apontando para ambiente `/dev`.

Solução: Garanta que a URL colada no frontend termina com /exec e o script esteja como 'Executar como Eu'.
