Manual de Deploy e Configuração

Bem-vindo ao guia oficial de publicação do sistema. Este manual foi estruturado passo a passo para garantir que você coloque o SaaS no ar de maneira segura e profissional, mesmo sem experiência prévia em nuvens corporativas.

💡 Dica Importante

Você pode usar as caixas de seleção interativas para marcar cada tarefa concluída. Seu progresso será mantido caso você recarregue a página!

1. Configurar o Repositório Git Local

O primeiro passo é clonar a base de código do projeto para o seu ambiente de desenvolvimento local utilizando o Git.

Terminal (PowerShell / Bash) 
# Clonar o repositório do Monorepo
git clone https://github.com/seu-usuario/protocolo-integrativo.git

# Acessar a pasta do projeto
cd protocolo-integrativo

3. Publicar o Backend no Google Apps Script

Vamos configurar o código do servidor no Apps Script e disponibilizar a API Web App.

⚠️ Atenção na Implantação

No painel de deploy do Web App, certifique-se de configurar Executar como: Eu (sua-conta) e Quem tem acesso: Qualquer pessoa (Anyone). Caso contrário, o frontend receberá erros de CORS ao tentar comunicar-se com a API.

4. Configurar e Publicar o Frontend

Configuração do arquivo de variáveis do Frontend e deploy no GitHub Pages.

frontend/src/shared/config/SystemConfiguration.js 
// Substitua pelo endereço gerado no Deploy do seu Apps Script Web App
export const SystemConfiguration = {
  API_BASE_URL: 'https://script.google.com/macros/s/SEU_DEPLOY_ID/exec',
  THEME: 'slate'
};

6. Testes e Procedimentos de Rollback

Diretrizes de contingência em caso de bugs críticos em produção.

🚨 Procedimento de Rollback do Servidor

Caso ocorra erro no backend após alterações no código:

1. Acesse o editor do Apps Script.
2. Vá em Implantar > Gerenciar Implantações.
3. Selecione a versão estável anterior na lista de edições e clique em Salvar.
A URL de produção continuará idêntica, mas executando o código seguro antigo em menos de 10 segundos.

7. FAQ & Resolução de Problemas

Q: Recebo erro "CORS Policy Blocked" no console do navegador. Como resolver?

A: Verifique se você configurou o Apps Script para permitir acesso de "Qualquer pessoa" (Anyone) no momento de implantar o Web App. Se o acesso estiver restrito apenas ao proprietário, o navegador bloqueará as chamadas.

Q: Os e-mails de alerta não estão chegando nos e-mails dos pacientes.

A: O Google Apps Script limita o envio diário de e-mails via MailApp (100 para contas gratuitas, 1500 para contas Google Workspace). Certifique-se de que a cota diária não foi excedida no console do Google Drive.
